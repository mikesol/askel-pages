Drive Indexing — The Core Problem

The Drive is almost entirely unindexed. Of all the documents referenced across skills, CLAUDE.md, and the playbook, only one — Varia's financial analysis sheet — is referenced by a permanent Drive file ID anywhere in the repo. Everything else is either undocumented, referenced by a vague description, or linked by a name that could resolve to multiple versions. That's exactly how the .docx mistake happened: searching by name returned the stale version.

Google Drive file IDs are permanent. A document can be renamed, moved to a different folder, or shared with new people — the ID stays the same. Skills that reference documents by ID can never accidentally grab the wrong version. Skills that don't, can.

The fix: a Drive registry

Before Phase 1 starts, create knowledge/drive-registry.md — a single file mapping every external document to its permanent ID, type, purpose, and canonical status. Each skill then cites documents by ID from this registry, never by name or search.

The pattern is already established for 1Password secrets (op://askel-server/...). Drive documents should follow the same discipline: a permanent pointer, not a name-based search. The playbook skills in particular need this — if playbook-dd says "fetch the staff interview form from Drive" without an ID, a future search for "staff interview" could return the wrong thing.

This registry also solves a subtler problem: when Varia creates a new operational document for Melers, it should be added to the registry so Steve can find it in future sessions without trawling the whole Drive.

Migration Phases

Phase 0 — Drive registry · 1 session · prerequisite

Create knowledge/drive-registry.md cataloguing every external document with its permanent ID before any skill migration starts. Without this, skills will be written with the same indexing gap that caused the .docx mistake.

  • One entry per document: name, ID, type (Doc/Sheet/Form/Folder), purpose, canonical status, last-verified date
  • Mark stale/superseded versions explicitly (playbook .docx → superseded by v2 EN)
  • Add to CLAUDE.md knowledge base index so it's always in context
  • Going forward: any new Drive document gets an entry before being referenced in a skill
